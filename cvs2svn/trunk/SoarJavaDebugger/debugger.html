Soar Debugger

Running the Software

  • Running on Windows:
    1. Download into any folder 
    2. Double-click on SoarDebuggerInJava.jar or run with "java -jar SoarDebugger.jar"
    3. Requires JDK 1.4 or later.
    4. Note -- it includes copies of required DLLs within the JAR file and will unpack them automatically if necessary into the current folder.
  • Running on Linux:
    1. Not tested on Linux yet.
    2. Download.
    3. Run with "java -jar SoarDebuggerInJava.jar"
    4. Requires JDK 1.4 or later.
    5. No support yet for automatic unpacking of required .so files

What's new in the latest release?

Version 4
-- Runs can be stopped
-- Soar runs in a separate thread from the UI so the UI remains responsive.  Can be switched with a flag.
-- Added button bar
-- Added multiple windows to the UI and made them basically functional
-- Added support for changing fonts

Version 3

-- Switched local kernel to use CreateInNewThread() so others can remotely connect into the debugger.
-- Turned off forced-auto install of libraries for now, so if they already exist they are left in place.

Version 2

-- Revised version based on SWT windowing toolkit not Swing (to make it more Eclipse compatible)
-- On Windows will auto-install the SWT libraries as well as the SML libraries
-- Layout manager logic not available -- just one trace window so far
-- When creating multiple agents in multiple windows cleanup is incomplete when shutting them down.

Version 1

-- First prototype using SML.
-- Remote menu works -- allows connections to remote instances of Soar (but only on same machine using default port)
-- Agent menu works -- allows creation of new agents, deletion of agents and selection of agents.
-- Redesigned to allow multiple frame windows to be open at once for a single Soar kernel instance

Major bugs in version 1:
-- The layout manager is probably broken right now (windows may appear but may not register correctly for events they need)
-- The only place to type commands is into the bottom of the "Trace" window.  The other windows will not update when Soar stops (or their output goes to the wrong window)
-- Running Soar for a long time and doing init-soar crashes the kernel (if you're using my included SML libraries) because I've yet to change the settings for my kernel's inline functions.
-- Trace output from remote processes and many commands is not pretty because newlines seem to be handled differently between local and remote kernels.
-- Error messages are minimal and most things are untested at this point.
